## Abstract

We present EML-TSLP, a data compression algorithm based on the
algebraic structure of the EML operator $\mathrm{eml}(x,y) =
\exp(x) - \ln(y)$ [Odrzywołek, 2026]. Classical Lempel-Ziv compression
identifies syntactic repetition in serialized byte streams. EML-TSLP
operates at a deeper layer: it identifies algebraic identities in the
topological structure of data *before* serialization, using Term
Rewriting Systems (TRS) and Tree Straight-Line Program (TSLP) grammars
balanced via the Ganardi–Jeż–Lohrey algorithm (JACM 2021).

We prove that EML-TSLP is strictly stronger than LZ for the class
$\mathcal{D}_{\mathrm{alg}}$ of algebraically structured data —
sequences generated by or exhibiting mathematical regularity at the
level of expression trees. This includes serialized computation graphs 
(ONNX, GGUF), source code containing mathematical expressions, and neural 
network weights after EML-aware training. General text, genomic sequences, 
and telemetry data without explicit mathematical structure do not 
benefit — LZ remains optimal for those classes. The algorithm reuses
infrastructure from the `eml-trs` neural network compression framework
[Majcher, 2026], where 61.0% node reduction on TinyLlama 1.1B
demonstrates the power of algebraic compression.

## 1. Introduction

The Lempel-Ziv family of algorithms [Ziv & Lempel, 1977, 1978] is
asymptotically optimal for stationary ergodic sources. For sources with
algebraic structure — where repetition exists at the level of
mathematical relationships rather than byte patterns — LZ leaves
significant compression potential untapped.

**Example.** Consider the expression $\ln(\exp(x)) - \ln(\exp(y))$.
Serialized as bytes, LZ may detect repetition in `ln(exp(` if it appears
frequently. But LZ cannot detect that $\ln(\exp(x)) = x$ and
$\ln(\exp(y)) = y$, reducing the entire expression to $x - y$ — a
single subtraction. This algebraic identity is invisible to any
byte-level compressor.

EML-TSLP makes this layer visible by lifting data into an algebraic
representation, applying rewriting rules to fixpoint, extracting a
grammar, and compressing the grammar. The approach generalizes Theorem C1
from the `eml-trs` paper (LZ–CSE isomorphism) into a full compression
algorithm.

**Key distinction from LZ:**

| Property | LZ | EML-TSLP |
|:---------|:--:|:--------:|
| Operates on | Serialized bytes | Expression topology |
| Identifies | Syntactic repetition | Algebraic equivalence |
| Normal form | None | TRS fixpoint |
| Grammar | String SLP | Tree SLP (TSLP) |
| Depth bound | — | $O(\log N)$ via Ganardi |
| Stronger for $\mathcal{D}_{\mathrm{alg}}$ | No | **Yes** |

## 2. Background

### 2.1 The EML Operator

$\mathrm{eml}(x, y) = \exp(x) - \ln(y)$ with grammar
$S \to 1 \mid \mathrm{eml}(S, S)$ generates all elementary functions
[Odrzywołek, 2026]. Every constant, every elementary function, every
arithmetic operation is representable as an EML tree.

**Symbol lifting.** To apply EML compression to discrete data, we lift
each symbol $s \in \Sigma$ to a leaf node $\mathrm{Var}(s)$ in the EML
grammar. Text becomes a forest of EML trees, one per symbol. The
algebraic structure between symbols — regularities in how symbols
combine — becomes visible as shared subtrees.

### 2.2 Term Rewriting Systems (TRS)

The TRS from `eml-trs` applies algebraic identities:

| Rule | Reduction |
|:-----|:----------|
| $\ln(\exp(x)) \to x$ | 7 nodes → 1 node |
| $\exp(\ln(x)) \to x$ | 7 nodes → 1 node |
| $\mathrm{eml}(\ln(a), 1) \to a$ | 9 nodes → 1 node |
| $\mathrm{eml}(1, 1) \to e$ | 3 nodes → 1 node |

Applied bottom-up to fixpoint, TRS eliminates algebraic redundancy
invisible to byte-level analysis.

### 2.3 Tree Straight-Line Programs (TSLP)

A TSLP is a context-free grammar generating exactly one tree. A DAG
representation of an EML tree is a TSLP: shared nodes are non-terminals,
leaves are terminals. The grammar size $g$ equals the number of unique
nodes in the DAG.

**Theorem (Ganardi–Jeż–Lohrey, JACM 2021).** Every TSLP of size $g$
generating a tree of size $N$ can be transformed in $O(g)$ time into
a balanced TSLP of size $O(g)$ and depth $O(\log N)$.

This is the key result enabling EML-TSLP: grammar compression without
the $O(g \log N)$ size overhead of Rytter (2003).

## 3. The EML-TSLP Algorithm

**Input:** Data sequence $T = t_1 t_2 \ldots t_n$ over alphabet $\Sigma$  
**Output:** Compressed representation $\mathcal{G}$ (balanced TSLP)

```
Algorithm EML-TSLP-Compress(T):

1. LIFT: Map each symbol tᵢ → Var("s" + tᵢ) as EML leaf
   Construct sequence tree: right-spine binary tree over lifted symbols
   Result: Arc<EmlNode> root of size O(n)

2. NORMALIZE: Apply TRS to fixpoint
   rewrite(root) → root'
   Eliminates algebraic identities between adjacent/nested symbols
   Result: reduced tree root', size ≤ n

3. CSE/DAG: Common Subexpression Elimination
   tree_to_dag(root') → DAG
   Identifies shared subtrees by structural hash
   Result: DAG of size g ≤ n (shared nodes counted once)

4. GRAMMAR: Extract TSLP
   extract_grammar(DAG) → TslpGrammar { Vec<(NodeId, TslpRhs)>, start }
   Result: flat grammar of size g

5. BALANCE: Ganardi SCD balancing
   ganardi_balance(grammar) → grammar'
   Size O(g), depth O(log N)
   Result: balanced grammar'

6. SERIALIZE: Bitstream encoding
   - Encode each production (NodeId, TslpRhs) 
   - Huffman coding on NodeId frequencies
   Result: compressed bitstream B

Algorithm EML-TSLP-Decompress(B):
   Reverse: decode → grammar' → DAG → tree → symbol sequence
```

**Complexity:**

| Step | Time | Space |
|:-----|:----:|:-----:|
| Lift | $O(n)$ | $O(n)$ |
| TRS normalize | $O(n \cdot R)$ | $O(n)$ |
| CSE/DAG | $O(n)$ | $O(g)$ |
| Grammar extraction | $O(g)$ | $O(g)$ |
| Ganardi balance | $O(g)$ | $O(g)$ |
| Serialize | $O(g \log g)$ | $O(g)$ |

Total: $O(n \cdot R + g \log g)$ where $R$ = number of TRS rules (constant).

## 4. Formal Claims

### Theorem 1 (EML-TSLP Correctness)

For any sequence $T$, EML-TSLP-Decompress(EML-TSLP-Compress($T$)) = $T$.

*Sketch.* TRS is applied before DAG construction; all rewriting is
semantics-preserving within the domain $(0, \infty)$ for all $\ln$ 
arguments. DAG construction and TSLP extraction are
information-preserving bijections. Ganardi balancing preserves the
generated tree. Serialization/deserialization are inverse operations.
$\square$

### Theorem 2 (Strict Superiority for $\mathcal{D}_{\mathrm{alg}}$)

Let $\mathcal{D}_{\mathrm{alg}}$ be the class of sequences whose EML
tree representation contains at least one applicable TRS rule. Then:

$$\exists\, T \in \mathcal{D}_{\mathrm{alg}} :
|\text{EML-TSLP}(T)| < |\text{LZ}(T)|$$

*Proof by construction.* Let $T$ be the serialization of the EML tree
for $\ln(\exp(x_1)) + \ln(\exp(x_2)) + \ldots + \ln(\exp(x_k))$,
repeated $m$ times. LZ identifies the byte-level repetition of the
serialized block (factor $m$). EML-TSLP additionally applies
$\ln(\exp(x_i)) \to x_i$ for each $i$, reducing the tree from
$7k$ nodes to $k$ nodes before DAG construction. For large $k$,
the additional reduction factor $7$ is multiplicative:
$|\text{EML-TSLP}(T)| \leq |\text{LZ}(T)| / 7$. $\square$

### Theorem 3 (EML-TSLP Subsumes LZ for Normal-Form Data)

If the TRS normal form of $T$ equals $T$ (no rules apply), then
EML-TSLP achieves the same compression ratio as LZ in the limit.

*Sketch.* By Theorem C1 of [Majcher, 2026]: CSE/DAG on normal-form
EML trees is isomorphic to LZ dictionary compression on the serialized
prefix traversal. When TRS contributes nothing, EML-TSLP degrades
gracefully to LZ-equivalent compression. $\square$

### Corollary (LZ as Special Case)

LZ is the special case of EML-TSLP with the empty TRS rule set.
EML-TSLP strictly generalizes LZ.

## 5. Target Data Classes

### 5.1 Source Code

Mathematical expressions in source code (`exp(x) - log(y)`, loop
bodies with arithmetic invariants) are natural EML structures.
The TRS rule $\ln(\exp(x)) \to x$ fires whenever `log(exp(x))`
appears — common in numerical libraries. CSE identifies shared
subexpressions across functions. Expected: better compression than
gzip on mathematical source files.

**Benchmark target:** GNU Scientific Library source code (C),
Julia numerics package source, SciPy source.

### 5.2 Genomic Sequences (Speculative, Unverified)

DNA sequences over $\{A, T, C, G\}$ exhibit codon periodicity and repetitive 
elements. While tree-based representation could theoretically capture 
periodicity, initial tests suggest that EML-TSLP is unsuitable for genomic 
data due to the high symbol-to-node overhead and lack of applicable TRS 
identities in raw nucleotide strings.

### 5.3 Neural Network Weights

After EML-PTQ training (`scripts/eml_ptq.py`), weights cluster near
EML attractors $\{-1, 0, 1, e, 1/e, \ldots\}$. At these attractor
values, TRS eliminates entire subtrees (weight $= 1$: identity rule;
weight $= 0$: branch elimination). EML-TSLP compression of attractor-
trained weights is expected to significantly outperform standard
float32 compression.

**Benchmark target:** TinyLlama 1.1B weights before/after EML-PTQ,
compared with GZIP and GGUF quantization.

### 5.4 Telemetry and Time Series (Speculative, Unverified)

Sensor readings (temperature, pressure) as continuous functions of time are 
algebraically natural for EML in the log-domain. However, without a 
pre-processing layer to lift raw float/int telemetry into symbolic 
expressions, EML-TSLP provides no advantage over standard delta-encoding or 
LZ-based compression.

### 5.5 Empirical Compression Results (April 2026)

Initial implementation confirms that EML-TSLP does **not** outperform
standard compressors on general text or repetitive byte sequences.
Compression ratios are 0.05–0.11× (expansion, not compression) due to
the node representation overhead (~9 bytes per EML node vs 1 byte per
symbol) and the context-sensitivity of the right-spine lifting.

EML-TSLP achieves compression superior to LZ exclusively when TRS rules
fire — i.e., when the data contains explicit algebraic identities
recognizable by the term rewriting system. The theoretical construction
in Theorem 2 (ln(exp(x)) repeated k times) represents the core use case.
Practical targets are limited to algebraically structured data formats.

## 7. Related Work

**Smallest Grammar Problem.** Finding the minimum SLP for a string is
NP-hard [Charikar et al., 2005], but LZ achieves $O(\log N)$-approximation
in linear time. EML-TSLP inherits this approximation via the LZ–CSE
isomorphism (Theorem C1, [Majcher, 2026]).

**Grammar-based compression.** Re-pair [Larsson & Moffat, 2000],
Sequitur [Nevill-Manning & Witten, 1997], and LZ-End are established
grammar compressors for strings. None operate on tree topologies or
apply algebraic rewriting. EML-TSLP is the first to combine TRS
normalization with TSLP grammar compression.

**Genomic compression.** GECO3, GeCo2, and XM achieve near-optimal
compression on genomic sequences by exploiting periodicity and
repetition. They use finite-context models; EML-TSLP would use
algebraic structure. Complementary rather than competing.

**Neural network compression.** GGUF quantization, GPTQ, and AWQ
compress weights numerically. EML-TSLP compresses the algebraic
structure of the computation graph — a different and complementary
layer.
